PM_SUMMARY:
- 드래프트 진행 중 입력 상태를 기준으로 포지션 한정 추천 5개와 설명 가능한 근거를 즉시 재계산해 보여주는 공개 반응형 웹 MVP를 구축한다.

ROLE_ASSIGNMENT_NOTES:
- 프론트엔드는 드래프트 입력 플로우, 포지션/진영 선택, 픽·밴 관리, 선호 챔피언 우선순위 입력, 추천 결과와 근거 표시를 담당하므로 필요합니다.
- 백엔드는 Data Dragon 기본 데이터 수집, 내부 보강 태그 데이터 관리, 룰·가중치 기반 추천 계산, 밴 제외 및 드래프트 변경 시 재계산 API를 담당하므로 필요합니다.
- 리뷰어는 추천 설명 가능성, MVP 범위 준수, 포지션 제한·밴 제외·시너지/카운터 우선 표기 같은 핵심 요구 충족 여부를 독립 점검해야 하므로 필요합니다.
- 검증자는 실제 드래프트 중간 단계 시나리오에서 입력 변경마다 추천 5개와 근거가 빠르게 갱신되는지 확인해야 하므로 필요합니다.

- 설계하라: MVP 사용자 플로우와 화면 구조를 포지션 선택→진영 선택→아군/상대 픽 입력→밴 입력→선호 챔피언 우선순위 입력→추천 확인 흐름으로 정의하고, scope는 반응형 단일 웹앱 화면 세트, output은 화면 맵·상태 전이·핵심 컴포넌트 목록, done criterion은 모든 input_requirements와 output_requirements가 화면 단에서 누락 없이 매핑된다.
- 구현하라: 드래프트 입력 UI를 블루/레드 선택, 현재까지 확정된 아군/상대 픽, 밴 목록, 사용자 포지션 선택 중심으로 구성하고, scope는 챔피언 검색/선택/해제와 중복 방지까지 포함, output은 상호작용 가능한 입력 화면, done criterion은 사용자가 드래프트 중간 단계에서도 필요한 상태를 모두 입력·수정할 수 있다.
- 구현하라: 선호 챔피언 다중 선택과 우선순위 조정 UI를 만들고, scope는 포지션 내 후보 기준 선택과 순위 변경 인터랙션, output은 선호 리스트 상태와 우선순위 값이 반영된 입력 모듈, done criterion은 여러 챔피언을 선택하고 우선순위를 명확히 조정할 수 있다.
- 구현하라: 추천 결과 영역을 추천 5개 카드 형식으로 만들고, scope는 챔피언 아이콘·이름·한줄 이유·시너지/카운터 우선 설명·기준별 평가 근거·선호 반영 배지 표시, output은 결과 카드 UI, done criterion은 acceptance_criteria의 설명 순서와 표시 항목이 카드마다 모두 보인다.
- 연결하라: 드래프트 상태 변경 시 추천을 즉시 재조회하는 프론트 상태 관리와 요청 흐름을 구성하고, scope는 픽·밴·포지션·선호 변경 이벤트 전부 포함, output은 재계산 트리거 로직과 로딩/에러 처리, done criterion은 사용자가 입력을 바꿀 때마다 최신 추천 결과가 자동 갱신된다.
